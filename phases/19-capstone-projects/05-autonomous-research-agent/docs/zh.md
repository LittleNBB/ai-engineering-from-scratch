# Capstone 05 — 自主研究 Agent（AI-Scientist 级别）

> Sakana 的 AI-Scientist-v2 发表了完整论文。Agent Laboratory 跑完了实验。Allen AI 分享了 trace。2026 年的形态是：对实验进行规划-执行-验证的树搜索、有预算约束的成本控制、沙箱化的代码执行、带视觉反馈的 LaTeX 写作器，以及自动化的 NeurIPS 风格评审集成。本毕设要求你构建一个这样的系统，在每篇论文 $30 的预算内端到端运行，并通过 Sakana 记录的沙箱逃逸红队测试。

**类型：** 毕业设计
**语言：** Python（Agent + 沙箱）、LaTeX（输出）
**前置课程：** Phase 2（机器学习）、Phase 3（深度学习）、Phase 7（Transformer）、Phase 10（从零构建 LLM）、Phase 14（Agent 工程）、Phase 15（自主系统）、Phase 16（多智能体）、Phase 18（安全与对齐）
**涵盖阶段：** P0 · P2 · P3 · P7 · P10 · P14 · P15 · P16 · P18
**时间：** ~40 小时

## 问题描述

自主研究 Agent 在 2026 年跨越了一个门槛。Sakana AI 的 AI-Scientist-v2 在 Nature 上发表，生成的论文通过了研讨会同行评审。ShinkaEvolve（ICLR 2026）将其扩展到假设进化。AMD 的 Agent Laboratory 发布了可复现的 trace。这些 Agent 不是魔法——它们是一个在候选实验树上运行的"规划-执行-验证"循环，配有成本上限、种子绑定的沙箱和自动评审。真正的功夫在于循环、预算和安全。

你将通过在一个窄领域中针对一个种子想法（例如，在 1 亿参数 Transformer 上进行注意力稀疏性消融实验）来实现这样一个系统，从而理解整个循环。价值不在于第一次运行就能发现新东西。价值在于基础设施：树搜索、实验沙箱、写作-评审循环、红队报告。Sakana 团队记录了沙箱逃逸失败案例；你的 Agent 必须通过同样的红队测试。

## 核心概念

Agent 是一个最优优先树搜索。节点是实验规格：（假设、配置、代码、预期结果）。展开步骤通过微小改动（替换优化器、调整批大小、消融某个组件）生成子节点。每个子节点在有硬性资源上限的新鲜沙箱中运行。结果反馈到评分函数中，按（新颖度 × 质量 × 剩余预算）对节点排序。树不断生长直到预算耗尽，然后对最佳分支进行论文撰写。

写作器是多模态的。它生成 LaTeX 草稿，编译，渲染图表，然后将渲染后的 PDF 反馈给 Claude Opus 4.7 的视觉模式，对排版、图表可读性和论点-证据对齐进行批评。一个由 5 个 LLM 评委组成的评审集成发出 NeurIPS 风格的评分（新颖度、严谨性、清晰度、可复现性、影响力）；如果平均分低于阈值，论文连同评审意见一起返回给写作器。

安全是承重结构。每个实验都在 E2B 或 Daytona 沙箱中运行，无网络出口、有墙上时间限制和固定的资源上限。Agent 的代码生成步骤通过策略层，阻止能逃逸沙箱的系统调用。红队报告复现了 Sakana 记录的攻击面（fork 炸弹、文件系统逃逸、LLM 编写的网络调用）。

## 架构

```
种子想法 + 领域
      │
      ↓
  文献检索（Semantic Scholar + OpenAlex + FAISS 缓存）
      │
      ↓
  LangGraph 规划-执行-验证树
      │
      ↓
  ┌── 展开节点 ────┐      每节点沙箱
  │                │      （E2B / Daytona）
  ↓                ↓      资源上限
  子节点_1       子节点_k   无网络出口
  │                │      确定性种子
  ↓                ↓
  运行实验         运行实验
  │                │
  ↓                ↓
  按（新颖度、质量、预算）对节点评分
      │
      ↓
  最佳分支 → LaTeX 写作器
      │
      ↓
  编译 + 视觉批评（Opus 4.7 视觉模式）
      │
      ↓
  评审集成（5 位 LLM 评委，NeurIPS 评分标准）
      │
      ↓
  paper.pdf + review.md + trace.json
```

## 技术栈

- 编排：LangGraph，支持检查点和人工审批门
- 树搜索：基于实验节点的自定义最优优先搜索（Sakana v2 的 AB-MCTS 风格）
- 沙箱：每个实验一个 E2B 实例，Docker-in-Docker 作为备选；通过 cgroups 限制资源
- 文献：Semantic Scholar Graph API + OpenAlex + 本地 FAISS 摘要缓存
- 写作器：LaTeX 模板 + Claude Opus 4.7（视觉模式），用于图表批评和排版
- 评审器：5 位评委集成（Opus 4.7、GPT-5.4、Gemini 3 Pro、DeepSeek R1、Qwen3-Max），加权聚合
- 实验框架：PyTorch 2.5 物理实验，W&B 日志记录
- 可观测性：Langfuse Agent 追踪，每篇论文 $30 硬性预算

## 构建步骤

1. **种子和领域界定。** 取一个种子想法（例如"研究 sub-1B Transformer 中注意力图的稀疏模式"）。定义搜索空间：模型、数据集、计算预算。

2. **文献检索。** 查询 Semantic Scholar + OpenAlex 获取 50 篇最高被引的相关论文；本地缓存摘要；生成 1 页领域摘要。

3. **树结构搭建。** 用种子假设初始化根节点。实现 `expand(node) -> children`，通过微小改动提案（每个子节点只改一个配置）。实现 `score(node)` 为加权的新颖度 × 质量 × 预算项。

4. **沙箱封装。** 每个实验运行 `docker run --network=none --memory=8g --cpus=2 --pids-limit=256 --read-only`（或等效的 E2B 策略）。种子写入沙箱；输出以只读方式挂载回主机。

5. **规划-执行-验证循环。** `plan` 提出子节点。`execute` 运行沙箱，捕获日志和指标。`verify` 对指标运行单元检查（损失是否下降？消融是否隔离了效果？）。失败节点在树上存储失败原因。

6. **写作器。** 预算耗尽后，选择最佳分支。用 matplotlib 渲染图表。将分支 trace 作为上下文，通过 Claude Opus 4.7 生成 LaTeX 草稿。编译。将编译后的 PDF 反馈给 Opus 4.7 视觉模式进行批评。迭代。

7. **评审集成。** 5 位评委按 NeurIPS 风格评分标准对草稿评分（新颖度、严谨性、清晰度、可复现性、影响力）。如果平均分 < 4.0/5，连同批评意见返回写作器。3 次重写后硬性停止。

8. **红队测试。** 构建或集成一组针对沙箱的对抗性任务：fork 炸弹、网络数据外泄尝试、文件系统逃逸、LLM 编写的 shell 元字符。确认全部被阻止。撰写发现报告。

9. **可复现性。** 每篇论文附带其树搜索 trace JSON、种子、W&B 运行链接、沙箱配置和一份端到端复现的 README。

## 交付物

`outputs/skill-ai-scientist.md` 是交付物。给定一个种子想法 + 一个领域 + $30 预算，运行完整管线，输出一篇可评审的论文和一个可复现性包。

| 权重 | 评分项 | 衡量方式 |
|:-:|---|---|
| 25 | 论文质量 | 对照已发表的研讨会论文进行盲审评分 |
| 20 | 实验严谨性 | 基线、种子、消融实验；每个论点都有结果表中的数据支撑 |
| 20 | 成本和计算纪律 | $30/篇论文的硬性上限，Langfuse 追踪 |
| 20 | 安全性 | 沙箱红队通过；网络策略和紧急停止机制已验证 |
| 15 | 可复现性 | 使用相同种子一键重跑可复现论文 |
| **100** | | |

## 练习

1. 在同一领域中对三个不同的种子想法运行管线。比较树搜索中哪些部分有重叠。识别被浪费的重复计算。

2. 在实验执行前为预估超过 $5 的节点添加人工审批门。衡量总成本降低了多少。

3. 将评审集成替换为单个评委。在一组已知质量差的论文保留集上衡量误接受率。

4. 引入网络数据外泄红队测试：Agent 编写代码尝试 `curl` 一个外部地址。确认 `--network=none` 策略阻止了该操作。记录尝试日志。

5. 将你的树搜索与随机扁平基线（相同预算，无展开策略）进行对比。报告新颖度 × 质量的提升。

## 关键术语

| 术语 | 业界说法 | 实际含义 |
|------|---------|---------|
| 树搜索 | "AB-MCTS 风格展开" | 基于新颖度×质量×预算评分对实验节点进行最优优先探索 |
| 沙箱 | "实验隔离" | 无网络、CPU/内存受限、种子固定、输入只读的容器 |
| 视觉批评 | "先渲染再阅读" | 将论文编译为 PDF，将 PDF 反馈给 VLM 进行排版和论点-证据批评 |
| 评审集成 | "自动同行评审" | 多个 LLM 评委按 NeurIPS 评分标准打分；加权聚合后决定管线走向 |
| 新颖度评分 | "这个新吗？" | 惩罚与 50 篇文献缓存相似度的启发式指标 |
| 成本上限 | "$ 预算" | 每篇论文总花费的硬性上限；Langfuse 计数器 + 运行前估算 |
| 红队 | "沙箱逃逸审计" | 如果策略有误就能逃逸沙箱的对抗性任务 |
