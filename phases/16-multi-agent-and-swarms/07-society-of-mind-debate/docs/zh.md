# 心智社会与多智能体辩论（Society of Mind and Multi-Agent Debate）

> Minsky 1986 年的前提——智能是专家的社群——每十年被重新发现一次。2023 年 Du 等人将其转化为具体算法：多个 LLM 实例提出答案，阅读彼此的答案，批评，然后更新。经过 N 轮，它们收敛到一个共识，在六个推理和事实性任务上超越了零样本 CoT 和自我反思。两个发现很重要：**多个智能体**和**多轮对话**各自独立地做出贡献。社群打败了单智能体独白；多轮交换打败了一次性投票。

**类型：** Learn + Build（学习 + 构建）
**语言：** Python（标准库）
**前置课程：** Phase 16 · 04（Primitive Model）
**时间：** ~60 分钟

## 问题

自我一致性（Self-consistency）——对一个模型采样多次并取多数答案——是你能附加的最便宜的推理改进。它有效，但很快饱和。你可以将样本翻倍却看不到另一个有意义的跳跃。

辩论打破了饱和。N 个智能体不是从一个模型独立采样，而是阅读彼此的推理并修订。样本之间的相关性降低了（它们不再是独立同分布的），而收敛点在独立同分布投票自信地错误的地方往往是正确的。

## 核心概念

### Du 等人 2023 年算法

来自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体中的每一个对问题产生一个初始答案。
2. 对于第 r = 2..R 轮：每个智能体被展示其他智能体第 r-1 轮的答案，并被要求"考虑这些答案，给出你更新后的答案。"
3. R 轮后，对最终答案进行多数投票。

论文在 MMLU、GSM8K、传记、MATH 和事实性基准上进行测试。辩论持续地优于 CoT 和自我反思。

### 两个独立旋钮

同一论文的消融实验：

- **仅智能体数量**（1 轮，N 个多数投票）在大多数任务上优于单智能体，但会达到平台期。
- **仅轮数**（1 个智能体看到自己先前的推理）几乎没有帮助——这是反思的已知弱点。
- **两者结合**产生大的跳跃。多个智能体之间的多轮交换驱动了增益。

### 为什么有效

两个机制：

1. **接触分歧。** 当一个智能体看到另一个智能体的推理链得出不同结论时，它必须要么辩护，要么更新。无论哪种方式，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **相关错误减少。** 在自我一致性中，所有样本来自同一个模型，因此错误是相关的——你平均出一个自信但错误的答案。不同模型或不同种子去相关。不同的*辩论观点*进一步去相关。

### 异质辩论（Heterogeneous Debate）

A-HMAD 和相关后续工作对不同智能体使用*不同的基础模型*。Llama + Claude + GPT 的辩论减少了单一种群崩溃（Lesson 26），因为一个模型家族的相关错误不会被其他模型家族共享。

缺点：参与辩论的弱模型可能将共识拖向其错误答案（参见"Should we be going MAD?"，arXiv:2311.17371）。

### NLSOM —— 129 智能体扩展

Zhuge 等人（"Mindstorms in Natural Language-Based Societies of Mind"，arXiv:2305.17066）将这个想法扩展到 129 成员的社群。结果：专业化和自组织在规模下涌现，系统在视觉问答等任务上超越了单智能体。

### 失败模式

- **谄媚级联（Sycophancy cascade）。** 所有智能体都顺从听起来最自信的那个智能体。辩论坍缩为最大声的声音。提示对抗性角色（"一个智能体必须反驳主流立场"）有帮助。
- **话题漂移。** 多轮辩论会偏离原始问题。缓解措施：每轮重新注入问题。
- **计算爆炸。** N 个智能体 × R 轮 = N·R 次 LLM 调用，每次的上下文都在增长。5 个智能体、5 轮辩论是 25 次不断增长上下文的调用。每个问题的成本可能超过单次 CoT 调用的 10 倍。

## 动手实现

`code/main.py` 在一个数学问题上运行 3 个智能体 × 3 轮辩论，每个智能体以不同（可能错误的）答案开始。智能体是脚本化的——每个通过按脚本化的置信度加权平均邻居的答案来"更新"。收敛在逐轮日志中可见。

演示展示了两个关键效果：

- 一轮交换使智能体更接近正确答案。
- 超过第 2 轮的额外轮次显示递减收益（匹配 Du 等人的平台期）。

运行：

```
python3 code/main.py
```

## 实际应用

`outputs/skill-debate-configurator.md` 为新任务配置辩论：智能体数量、轮数、异质性（相同模型 vs 混合）、角色分配（对称 vs 一个对抗者）。它还会在运行前估算 token 成本。

## 交付建议

如果你要交付辩论系统：

- **将轮数限制在 3。** Du 等人表明 3 轮捕获了大部分增益。更多是成本，不是质量。
- **将智能体数限制在 5。** 超过 5 个，上下文膨胀和成本占主导。
- **默认异质。** 池中至少两个不同的基础模型。
- **对抗性席位。** 一个智能体被提示无论怎样都不同意。打破谄媚。
- **记录每轮。** 隐藏中间轮次的辩论系统无法被调试或审计。

## 练习

1. 运行 `code/main.py`，然后将轮数设为 5，观察递减收益。在第几轮额外收敛停止？
2. 添加第四个智能体扮演对抗性角色：总是不同意当前多数。这会破坏还是改善收敛？
3. 绘制（打印）每轮的一致分数（多数答案上的智能体比例）。它何时达到 1.0，这等价于"正确"吗？
4. 阅读 Du 等人第 4 节消融实验。使用此代码复现"仅智能体" vs "仅轮数" vs "两者结合"的结果。
5. 阅读"Should we be going MAD?"（arXiv:2311.17371）并列出轮询之外的两种辩论变体——例如法官主导、辩论链、对抗式。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Society of Mind（心智社会） | "Minsky 的想法" | 智能是交互的专家；1986 年的框架现在通过 LLM 辩论实现。 |
| Multi-agent debate（多智能体辩论） | "智能体争论" | N 个智能体提出、互相批评、在 R 轮中修订、多数投票。 |
| Consensus（共识） | "它们达成一致" | 不是认识论真相——只是多数答案上的比例。可能是自信地错误。 |
| Rounds（轮次） | "交换步骤" | 一轮 = 每个智能体阅读其他智能体并更新一次。 |
| Heterogeneous debate（异质辩论） | "混合模型家族" | 使用不同的基础模型来去相关错误。 |
| Sycophancy cascade（谄媚级联） | "所有人都同意声音最大的那个" | 辩论失败，智能体不顾正确性都顺从最自信的智能体。 |
| NLSOM | "129 智能体社群" | 基于自然语言的心智社会；Zhuge 等人的扩展版本。 |
| Correlated error（相关错误） | "同一个模型，同一个 bug" | 为什么自我一致性会饱和；跨不同观点的辩论去相关。 |
