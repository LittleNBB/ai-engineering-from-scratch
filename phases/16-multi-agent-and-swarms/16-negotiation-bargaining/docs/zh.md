# 谈判与议价（Negotiation and Bargaining）

> 智能体就资源、价格、任务分配和条款进行谈判。2026 年的基准测试集很明确：NegotiationArena（arXiv:2402.05863）表明 LLM 可以通过人格操控（"绝望"）提升约 20% 的收益；"Measuring Bargaining Abilities"（arXiv:2402.15813）表明买方比卖方更难，规模也没有帮助——他们的 **OG-Narrator**（确定性报价生成器 + LLM 叙述者）将成交率从 26.67% 推到 88.88%；大规模自主谈判竞赛（arXiv:2503.06416）运行了约 18 万次谈判，发现**隐藏思维链**的智能体通过向对手隐藏推理来获胜；Bhattacharya 等人 2025 年在哈佛谈判项目指标上对 Llama-3 评为最有效、Claude-3 最激进、GPT-4 最公平。本课实现合同网协议（FIPA 祖先，Lesson 02），连接一个 LLM 风格的买方/卖方，运行 OG-Narrator 风格的分解，并测量成交率如何随每个结构选择变化。

**类型：** Learn + Build（学习 + 构建）
**语言：** Python（标准库）
**前置课程：** Phase 16 · 02（FIPA-ACL Heritage），Phase 16 · 09（Parallel Swarm Networks）
**时间：** ~75 分钟

## 问题

两个智能体需要就价格达成一致。在纯语言提示词下自行谈判，2024-2026 年的 LLM 以令人惊讶的低速率达成交易（在严格参数化的议价中约 27%，见 arXiv:2402.15813）。规模也不能修复：GPT-4 在谈判上并不比 GPT-3.5 有结构性优势；它只是在谈判的*语言*上更好。

根本问题是 LLM 混淆了两项工作——决定报价和叙述报价。OG-Narrator 分离了这些：确定性报价生成器计算数值变动；LLM 只负责叙述。成交率跳到约 89%。

这反映了经典的多智能体发现：将机制与通信层解耦是制胜之道。合同网协议（Contract Net Protocol，FIPA，1996；Smith，1980）是参考的任务市场机制。将 LLM 插入叙述槽，你就得到了一个现代 LLM 驱动的任务市场。

## 核心概念

### 合同网，一段话概括

Smith 1980 年的合同网协议：一个**管理者**广播一个**征集提案（cfp）**；**竞标者**回复包含其报价的 **propose** 消息；管理者选择赢家并向赢家发送 **accept-proposal**，向输家发送 **reject-proposal**。赢家执行工作。可选消息：**refuse**（竞标者拒绝提案）。FIPA 将其编纂为 `fipa-contract-net` 交互协议。

### 为什么 OG-Narrator 获胜

"Measuring Bargaining Abilities of Language Models"（arXiv:2402.15813）观察到：

- LLM 经常违反议价规则（以荒谬的价格报价，忽略对方的 ZOPA）。
- 锚定效果差（接受糟糕的首次报价；以象征性而非策略性的金额还价）。
- 仅靠规模不能修复这些。更大的模型产生更合理的语言，但策略错误相似。

OG-Narrator 的分解：

```
           ┌──────────────────┐        ┌──────────────────┐
  状态  →  │ 报价生成器       │ 价格 → │  LLM 叙述者       │ → 消息
           │  （确定性）      │        │  （撰写           │
           │                  │        │   人类风格的       │
           └──────────────────┘        │   伴随文字）      │
                                       └──────────────────┘
```

报价生成器是经典的谈判策略：鲁宾斯坦议价模型、Zeuthen 策略或简单的以牙还牙价格策略。LLM 负责叙述。消息包含确定性价格和自然语言框架。

成交率跳升是因为：
- 价格保持在议价区域内。
- 锚点是策略性的，而非情绪化的。
- LLM 做它擅长的事：写作。

### NegotiationArena 发现

arXiv:2402.05863 提供了权威基准。主要发现：

- LLM 可以通过采用人格（"我急切地想在周五前卖掉这个"）来提升约 20% 的收益——人格操控是一种真实的策略。
- 公平/合作型智能体被对抗性智能体利用；防御需要显式的反姿态。
- 对称配对在约 40% 的基准场景中收敛到不公平的结果。

这不是"LLM 是糟糕的谈判者"。而是"LLM 谈判太像人类，包括可被利用的部分"。

### 思维链隐藏

大规模自主谈判竞赛（arXiv:2503.06416）在多种 LLM 策略上运行了约 18 万次谈判。赢家向对手隐藏了推理：

- 如果智能体将"我只会出到 75 美元；我的保留价是 70 美元"打印到公开可见的草稿纸上，对手就会读到它。
- 赢家私下计算策略；输出通道只包含报价和最低必要的叙述。

这是经典博弈论（Aumann 1976 关于理性和信息）在 2026 年的回响：泄露你的私人估值会损失收益。LLM 不会直觉理解这一点，会愉快地在推理跟踪中输入保留价，而这些跟踪对对手可见。

工程要点：将私人草稿纸上下文与公开消息上下文分离。不可选。

### Bhattacharya 等人 2025 — 模型排名

在哈佛谈判项目指标（原则性谈判、BATNA 尊重、利益互惠）上：

- **Llama-3** 在达成交易方面最有效（成交率 + 收益）。
- **Claude-3** 是最激进的谈判者（高锚点、晚让步）。
- **GPT-4** 是最公平的（配对间收益方差最小）。

这是 2025 年的快照。重点不是 2026 年 4 月哪个模型获胜——而是不同基础模型有持续的谈判风格。异质集成（Lesson 15）将此作为多样性来源。

### 通过合同网 + LLM 进行任务分配

合同网在 LLM 多智能体中的现代复用：

1. 管理者智能体将任务分解为单元。
2. 向工人智能体广播带任务描述的 `cfp`。
3. 每个工人返回一个报价：`(price, eta, confidence)`，其中 price 可以是 token、计算单元或美元。
4. 管理者选择赢家（单个或多个，取决于任务）并授予。
5. 被拒绝的工人可以自由竞标其他任务。

这可以很好地扩展到超过 100 个工人，因为协调是广播-响应，而非同步聊天。在生产中使用：Microsoft Agent Framework 的编排模式，一些 LangGraph 实现。

### LLM 利益相关者交互谈判

NeurIPS 2024 引入了带有**秘密分数**和**最低接受阈值**的多方可评分博弈。每个利益相关者有私人效用；LLM 必须从消息中推断它们。这是双方议价到 N 方联盟形成的泛化。对于具有异质工人能力的生产任务市场相关。

### 叙述 vs 机制规则

在所有 2024-2026 年谈判基准中，一致的工程规则是：

> 让 LLM 叙述。不要让 LLM 计算报价。

如果报价需要是数字（价格、ETA、数量），从谈判状态确定性地生成它，让 LLM 产生框架。如果报价需要是提案结构（任务分解、角色分配），让 LLM 起草，但在发送前对照 Schema 验证和约束检查。

## 动手实现

`code/main.py` 实现了：

- `ContractNetManager`、`ContractNetTask`、`Bid` — 管理者 + 竞标者，广播 cfp，收集提案，授予。
- `og_narrator_bargain(state, rng)` — OG-Narrator 买方：确定性 Zeuthen 风格向中点让步。
- `seller_response(state, rng)` — 确定性卖方还价策略（两种风格的结构性基准真相）。
- `naive_llm_bargain(state, rng)` — 模拟全 LLM 议价者：高方差选价，经常超出 ZOPA。
- 测量：每次试验采样新保留价，1000 次试验的成交率。

运行：

```
python3 code/main.py
```

预期输出：朴素 LLM 成交率约 65-75%；OG-Narrator 成交率约 85-95%；15-25 个百分点的差距是将报价生成与叙述分解的结构性优势。加上一个合同网任务市场分配示例，三个竞标者和一个任务。

## 实际应用

`outputs/skill-bargainer-designer.md` 设计议价协议：谁生成报价（确定性还是 LLM）、谁叙述、私人草稿纸如何与公开消息分离、以及成交率如何监控。

## 交付建议

生产议价检查清单：

- **分离草稿纸。** 私人状态永远不进入对手的上下文。这是不可协商的。
- **确定性报价生成。** 价格、数量、ETA：计算，不要提示。
- **验证所有传入报价**对照 Schema。在协议边界拒绝超出 ZOPA 的报价。
- **限制轮次。** 最多 3-5 轮；僵局时升级到调解者。
- **持续测量成交率和收益方差。** 成交率下降是症状——通常是提示词漂移或对手端攻击。
- **记录所有被拒绝的提案**及其确定性理由。对于合同网管理者，落选的竞标者需要理解原因。

## 练习

1. 运行 `code/main.py`。确认 OG-Narrator 在成交率上击败朴素 LLM。差距多大？
2. 实现**基于人格的收益提升**（arXiv:2402.05863）——买方仅在叙述中采用"本周急需购买"的人格，报价生成器不变。成交率或收益会变化吗？
3. 实现思维链**隐藏**：维护一个不传递给对手的私人草稿纸字符串。如果你意外泄露了它会怎样（通过交换通道模拟）？
4. 将合同网扩展为有保留价的 N 竞标者拍卖。当所有出价都超过保留价时，管理者如何在最低价和最高质量之间决定？你选择哪种授予规则，为什么？
5. 阅读 Bhattacharya 等人 2025 年关于哈佛谈判项目指标的论文。实现两个不同风格（激进 vs 公平）的议价者。测量对称和非对称配对下的收益方差。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Contract Net（合同网） | "任务市场" | Smith 1980，FIPA 1996。cfp + propose + accept/reject。经典的任务市场。 |
| ZOPA（可能协议区间） | "可能达成协议的区域" | 买方最高价和卖方最低价之间的重叠。超出此范围的报价无法成交。 |
| BATNA（最佳替代方案） | "谈判协议的最佳替代" | 如果交易失败你的后备方案。设定你的保留价。 |
| OG-Narrator | "报价生成器 + 叙述者" | 分解：确定性报价，LLM 叙述。 |
| Zeuthen 策略 | "风险最小化让步" | 基于风险限制进行让步的经典报价生成器。 |
| Rubinstein 议价 | "交替报价均衡" | 带贴现的无限视界议价的博弈论模型。 |
| CoT 隐藏（思维链隐藏） | "隐藏你的推理" | arXiv:2503.06416 中的赢家保持私人草稿纸；公开通道只显示报价。 |
| Persona manipulation（人格操控） | "情绪姿态" | arXiv:2402.05863：通过绝望/紧急人格获得约 20% 的收益增益。 |
