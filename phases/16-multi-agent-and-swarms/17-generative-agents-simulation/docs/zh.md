# 生成式智能体与涌现仿真（Generative Agents and Emergent Simulation）

> Park 等人 2023（UIST '23，arXiv:2304.03442）用三部分架构填充了 **Smallville**——一个拥有 25 个智能体的沙盒：**记忆流**（自然语言日志）、**反思**（智能体对自己流的更高层综合）和**计划**（日级行为，然后是子计划）。标志性结果是情人节派对的涌现：一个被种下"想举办情人节派对"的智能体，在没有进一步脚本的情况下，产生了通过人群传播的邀请、协调了日期，派对最终举行——从 24 个最初对此毫不知情的智能体开始。消融实验表明三个组件都是可信度所必需的。有据可查的失败是空间规范错误（进入已关闭的商店、共用单人浴室）。这是 2026 年智能体仿真和多智能体社会评估的参考架构。

**类型：** Learn + Build（学习 + 构建）
**语言：** Python（标准库）
**前置课程：** Phase 16 · 04（Primitive Model），Phase 16 · 13（Shared Memory）
**时间：** ~75 分钟

## 问题

大多数多智能体系统是严格脚本化的团队：规划者规划、编码者编码、审阅者审阅。这对定义明确的任务有效。它不能捕获当智能体拥有记忆、优先级和开放世界时产生的涌现的、非脚本化的行为。研究、社会仿真以及越来越多的游戏 AI 需要这第二种。

Smallville 架构是其基准。在 Park 2023 之前，最好的智能体仿真都是浅层脚本跟随者；之后，该模式成为开放世界中生成式智能体的默认方案。如果你在 2026 年构建智能体仿真，你要么使用 Smallville 的三个组件，要么明确说明为什么不用。

## 核心概念

### 三个组件

**记忆流（Memory Stream）。** 一个仅追加的观察、行动、反思和计划日志。每个条目有时间戳、类型、描述（自然语言），以及派生元数据：**新近度（recency）**、**重要性（importance）**（智能体自评 1-10 分）和**相关性（relevance）**（与当前查询的余弦相似度）。

```
[2026-02-14 09:12:03] 观察: Isabella Rodriguez 问我是否喜欢爵士乐
[2026-02-14 09:14:22] 反思:   我喜欢关于音乐的长对话
[2026-02-14 10:05:00] 计划:   今晚参加 Isabella 的情人节派对
```

记忆检索组合三个分数：`score = w_recency * e^(-decay * age) + w_importance * importance + w_relevance * cos_sim`。Top-k 条目进入当前提示词。

**反思（Reflection）。** 定期地（每 N 条记忆或在重要事件时），智能体从近期记忆生成更高阶的综合。反思条目回到流中，像任何其他记忆一样可检索。这就是智能体如何构建"理解"——架构对长期信念的等价物。

**计划（Plan）。** 自顶向下的分解。首先，日级计划的大致轮廓（"去上班，和 Klaus 一起吃晚饭"）。然后小时级计划。然后行动级计划。计划是可修订的：当观察与计划矛盾时，智能体重新规划受影响的段落。

### 为什么三个都重要（消融）

Park 等人分别去除观察、反思和计划进行了消融。每个消融都损害可信度：

- 没有**观察**，智能体错过上下文，基于过时的信念行动。
- 没有**反思**，智能体无法形成高阶信念；交互停留在浅层。
- 没有**计划**，行为变成反应式噪音；目标消散。

来自人类评分者的可信度分数在三个组件齐全时最高；去掉任何一个都会产生可测量的退化。

### 情人节涌现

一个智能体 Isabella Rodriguez 被种下目标"想在 2 月 14 日下午 5 点在 Hobbs Cafe 举办情人节派对"。其他 24 个智能体没有收到这样的种子。在模拟天数中：

1. Isabella 的计划包括邀请人。
2. 每次邀请成为邻居记忆流中的一个观察。
3. 那个邻居的反思产生信念："Isabella 在举办派对。"
4. 邻居的计划纳入"2 月 14 日参加派对"。
5. 邻居告诉其他邻居。邀请在没有中央协调的情况下传播。
6. 2 月 14 日下午 5 点，多个智能体聚集在 Hobbs Cafe。

这是技术意义上的涌现：系统级行为（派对）从局部交互（双边邀请 + 个体计划）中产生，没有中央编排器。

### 有据可查的失败模式

Park 等人明确记录了：

- **空间规范错误。** 智能体走进已关闭的商店。智能体试图共用同一间单人浴室。智能体在不适合用餐的房间用餐。模型不能仅从环境中推断社会物理规范。
- **记忆溢出。** 深度仿真运行导致记忆检索成本增长。实际补救：定期记忆压缩（摘要-修剪）和低重要性条目的衰减。
- **反思幻觉。** 反思可能编造记忆流中不存在的关系。缓解：在反思提示词中包含源记忆 ID，并在检索时验证。

这些是生产相关的失败模式：任何 2026 年智能体仿真都会继承它们。

### 三组件实现规则

1. **记忆是仅追加的。** 永不修改记忆条目。纠正是新条目。
2. **重要性评分是廉价的。** 在写入时调用 LLM 评分 1-10。缓存分数。
3. **检索是排序的，不是过滤的。** 按组合分数 top-k；不使用硬过滤器（会丢失上下文）。
4. **反思定期运行。** 当未处理记忆的重要性总和超过阈值（如 150）时触发。
5. **计划是可修订的。** 当新观察与计划矛盾时，只重新生成受影响的段落，而非整个计划。

### 超越 Smallville 的生成式智能体

2024-2026 年的后续文献扩展了该架构：

- **用于政策/市场研究的多智能体社会仿真。** 类 Smallville 的人群模拟用户对功能的响应行为。比 A/B 测试更快；准确性有争议。
- **游戏中的 NPC AI。** 使用 Smallville 智能体的 RPG 产生涌现的故事情节，而非脚本化的任务。
- **生成式智能体评估基准。** 指标不是任务准确率，而是长运行中行为的可信度 + 连贯性。

架构是参考。扩展替换组件（向量存储替代记忆、检索增强反思、神经符号计划）但保持三部分结构。

### 为什么这对多智能体工程很重要

Smallville 是多智能体现实在组件正确时成本低廉的概念证明。该架构现在已在开源模型上复现（较小的 LLM 优雅地而非急剧地降低可信度）。任何需要**涌现社会行为**的生产系统都使用这种形态。任何需要**紧密任务执行**的系统都使用本阶段前面的监督者/角色/原语模式。

## 动手实现

`code/main.py` 用标准库 Python 实现了三个组件，使用脚本化智能体策略（无真实 LLM）。演示在微型版本中复现了情人节派对涌现：

- `MemoryStream` — 带新近度/重要性/相关性检索的仅追加日志。
- `reflect(stream)` — 对近期高重要性记忆的脚本化反思。
- `plan(agent_state)` — 基于当前信念的日级和小时级计划。
- 场景：5 个智能体。智能体 1 以"下午 5 点举办派对"开始。在模拟时间片中，邀请传播，智能体聚集。

运行：

```
python3 code/main.py
```

预期输出：逐时间片跟踪。到最后一个时间片，5 个智能体中至少 3 个在计划中显示派对，并聚集在派对地点。单个种子在没有任何编排器的情况下产生了协调到达。

## 实际应用

`outputs/skill-simulation-designer.md` 设计生成式智能体仿真：智能体数量、记忆 Schema、反思节奏、计划视界和评估指标。

## 交付建议

生产仿真规则：

- **记忆就是数据库。** 在规模上选择真正的存储（向量数据库、Postgres）。内存中的标准库仅用于原型。
- **记录检索跟踪。** 对每个行动，记录驱动它的 top-k 记忆。这是你的调试能力。
- **预算每个智能体的 token。** 每个智能体每时间片的检索 + 反思 + 计划是 O(k) 次 LLM 调用。N 个智能体 × T 个时间片 × 每时间片调用次数可能远超你的预算。
- **定期压缩记忆。** 摘要-修剪低重要性条目。保留策略是设计决策，不是细节。
- **显式检测空间/社会规范违规。** 架构不会学习它们。

## 练习

1. 运行 `code/main.py`。确认 3+ 个智能体聚集在派对。将智能体增加到 10——涌现是否仍然发生？
2. 去除反思步骤。行为看起来怎样？映射到 Park 2023 的消融发现。
3. 引入一个竞争的种子目标（"Klaus 想在下午 5 点做一个研究报告"）。智能体是分裂，还是一个目标主导？什么决定了这一点？
4. 添加空间约束：Hobbs Cafe 最多容纳 4 个智能体。仿真是否优雅地处理溢出，还是会碰到"单人浴室"失败模式？
5. 阅读 Park 等（arXiv:2304.03442）第 6 节（涌现行为实验）。识别一个在你的微型版本中不可复现的行为。你需要增强架构的哪个组件？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Memory stream（记忆流） | "智能体的日记" | 观察、行动、反思、计划的仅追加日志。 |
| Recency（新近度） | "记忆有多新" | 按年龄的指数衰减分数。 |
| Importance（重要性） | "智能体有多在意" | 写入时自评 1-10。已缓存。 |
| Relevance（相关性） | "与当前查询有多相关" | 余弦相似度（基于嵌入）。 |
| Reflection（反思） | "高阶信念" | 从近期记忆生成的综合，作为新记忆重新摄入。 |
| Plan（计划） | "日/小时/行动分解" | 自顶向下的计划树。观察矛盾时可修订。 |
| Smallville | "Park 2023 的沙盒" | 产生了情人节涌现的 25 智能体仿真。 |
| Believability（可信度） | "质量指标" | 人类评分者对行为是否像合理智能体的评分。 |
