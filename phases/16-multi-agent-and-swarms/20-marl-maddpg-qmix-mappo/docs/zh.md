# 多智能体强化学习 — MADDPG、QMIX、MAPPO（MARL — MADDPG, QMIX, MAPPO）

> 多智能体协调的强化学习遗产，在 2026 年仍然为 LLM 智能体系统提供信息。**MADDPG**（Lowe 等，NeurIPS 2017，arXiv:1706.02275）引入了集中训练、分散执行（CTDE）：每个批评者在训练期间看到所有智能体的状态和动作；测试时只运行本地行动者。适用于合作、竞争和混合场景。**QMIX**（Rashid 等，ICML 2018，arXiv:1803.11485）是带单调混合网络的价值分解；每个智能体的 Q 值组合为联合 Q，使 `argmax` 能干净地分配——在星际争霸多智能体挑战（SMAC）上占主导。**MAPPO**（Yu 等，NeurIPS 2022，arXiv:2103.01955）是带集中价值函数的 PPO；在粒子世界、SMAC、Google Research Football、Hanabi 上以最少调优"出奇地有效"。这些为必须分散行动的智能体团队的训练策略奠定了基础。MAPPO 是 **2026 年合作 MARL 的默认基线**。本课在一个小型网格世界玩具上从零构建每个算法，在接触 LLM 智能体训练之前将这三个概念刻入肌肉记忆。

**类型：** Learn（学习）
**语言：** Python（标准库，无 NumPy 的小型实现）
**前置课程：** Phase 09（Reinforcement Learning），Phase 16 · 09（Parallel Swarm Networks）
**时间：** ~90 分钟

## 问题

LLM 智能体系统越来越多地训练智能体间协调的策略：何时让步、何时行动、调用哪个对等方。告诉你如何训练此类策略的文献是多智能体强化学习（Multi-Agent Reinforcement Learning，MARL），它早于 LLM 浪潮，有一小组主导算法。

没有模式词汇表阅读 MARL 论文是痛苦的。集中训练分散执行（CTDE）、价值分解和集中批评者不是流行词——它们是对特定问题的特定回答：

- 独立 RL（每个智能体独自学习）从每个智能体的角度看是非平稳的。糟糕。
- 集中 RL（一个智能体控制所有）不扩展且违反执行约束。
- CTDE 两全其美：用全局信息训练，用本地策略部署。

## 核心概念

### 论文使用的三个环境

- **粒子世界（多智能体粒子环境）。** 简单的 2D 物理，带合作/竞争任务。MADDPG 的原始测试床。
- **星际争霸多智能体挑战（SMAC）。** 合作微操，部分观测。QMIX 的测试床。离散动作，连续状态。
- **Google Research Football、Hanabi、MPE。** MAPPO 基线。

不同环境有不同的动作/观测类型。算法相应选择。

### MADDPG（2017）— CTDE 模式

每个智能体 `i` 有一个行动者 `mu_i(o_i)`，将其自身观测映射到动作。每个智能体还有一个批评者 `Q_i(x, a_1, ..., a_n)`，在训练期间看到所有观测和所有动作。行动者通过策略梯度根据批评者的评估进行更新。

```
行动者更新:   grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) at a_i=mu_i(o_i)]
批评者更新:   对 Q_i(x, a_1..n) 进行 TD，给定下一状态的联合估计
```

为什么 CTDE：在训练时，我们知道每个人的动作；我们用它来减少每个批评者的方差。在部署时，每个智能体只看到 `o_i` 并调用 `mu_i(o_i)`。

失败模式：批评者随 N 个智能体增长（输入包含所有动作）。没有近似的情况下不能扩展到约 10 个智能体以上。

### QMIX（2018）— 价值分解

仅限合作。全局奖励是每个智能体 Q 值的单调函数之和：

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证 `argmax_a Q_tot` 可以通过每个智能体独立选择 `argmax_{a_i} Q_i` 来计算。这**正是你需要的分散执行属性**。在训练时，一个混合网络从每个智能体的 Q 值产生 `Q_tot`。

为什么 QMIX 在 SMAC 上胜出：合作星际争霸微操有同质智能体、局部观测、全局奖励——完美适合价值分解。

失败模式：单调性约束是限制性的；有些任务的奖励结构不是单调可分解的（一个智能体为团队牺牲）。扩展（QTRAN、QPLEX）放松了这一点。

### MAPPO（2022）— 被忽视的默认

多智能体 PPO：带集中价值函数的 PPO。每个智能体有自己的策略；所有智能体共享（或有每个智能体的）能看到完整状态的价值函数。Yu 等人 2022 年在五个基准上将 MAPPO 与 MADDPG、QMIX 及其扩展进行了基准测试，发现：

- MAPPO 在粒子世界、SMAC、Google Research Football、Hanabi、MPE 上匹配或击败离策略 MARL 方法。
- 最少的超参数调优需求。
- 训练稳定；跨种子可复现。

社区在此论文之前低估了在策略 MARL。2026 年，MAPPO 是合作 MARL 的默认基线；任何新方法都必须击败它。

### 为什么 LLM 智能体工程师应该关心

三个直接用途：

1. **路由器训练。** 一个元智能体选择哪个子智能体处理任务。这是一个有 N 个分散子智能体和一个集中路由器的 MARL 问题。MAPPO 适合。
2. **角色涌现。** 在生成式智能体仿真中，训练智能体随时间采用互补角色是一个伪装的 MARL 问题。QMIX 风格的价值分解通过构造强制互补性。
3. **多智能体工具使用。** 当智能体共享工具并竞争预算时，通过 CTDE 训练它们产生可部署的本地策略，尊重资源约束。

实际告诫：2026 年，大多数生产 LLM 智能体系统通过提示词而非训练来获取策略。当你有（a）大量交互数据、（b）清晰的奖励信号、（c）投资训练基础设施的意愿时，MARL 才会出现。

### CTDE 作为超越 RL 的设计模式

即使不训练，CTDE 也是一个有用的架构模式：

- 在*设计*期间，假设全团队可见性。
- 在*运行时*，强制分散执行：每个智能体只看到 `o_i`。

该模式迫使你保持每个智能体的状态显式，并提前考虑部分可观测性。许多生产多智能体系统隐式地假设到处都有共享状态——CTDE 纪律防止了这一点。

### 非平稳性问题

当多个智能体同时学习时，每个智能体的环境（包含其他智能体的策略）是非平稳的。经典单智能体 RL 证明失效。本课中的所有 MARL 算法都解决了这个问题：

- MADDPG：全局批评者看到所有动作，因此其价值估计是平稳的。
- QMIX：价值分解将学习转移到联合 Q 空间，其中最优性是良定义的。
- MAPPO：集中价值函数抑制了其他策略变化带来的方差。

在 LLM 智能体系统中，非平稳性表现为"我的智能体上个月工作正常，现在上游的其他智能体变了，我的就出问题了"。用 CTDE 训练 MARL 是有原则的修复；提示词级的修复更快但不持久。

### 本课不涵盖的内容

训练实际网络是 Phase 09 的话题。本课构建脚本化策略版本，展示 CTDE、价值分解和集中价值模式，不进行梯度更新。目标是在你拿起完整 MARL 库（PyMARL、MARLlib、RLlib multi-agent）之前内化这些模式。

## 动手实现

`code/main.py` 实现了三种模式演示，都在一个 2 智能体合作小型网格世界上：

- 环境：2 个智能体在 4x4 网格上，一个奖励颗粒。任何智能体到达颗粒时奖励 = 1；任务完成。
- `IndependentAgents` — 每个智能体将其他智能体视为环境。基线。
- `MADDPGStyle` — 集中批评者计算联合价值；行动者策略从中更新。脚本化策略改进。
- `QMIXStyle` — 带单调混合器的价值分解。
- `MAPPOStyle` — 集中价值函数；策略针对共享基线更新。

四种都运行相同的 episode 并报告平均到达目标步数。CTDE 变体收敛到比独立基线更短的路径。

运行：

```
python3 code/main.py
```

预期输出：独立智能体平均约 6 步；CTDE 变体收敛到约 3.5 步（4x4 网格的最优是 3）。尽管是脚本化策略，模式差异仍然显现。

## 实际应用

`outputs/skill-marl-picker.md` 是一个技能，为给定多智能体任务选择 MARL 算法：合作 vs 竞争、同质 vs 异质、动作空间类型、规模、奖励信号。

## 交付建议

MARL 在生产中很少见。当你使用时：

- **从 MAPPO 开始。** 2022 年论文将其确立为基线；先复现它可以节省追逐更花哨方法的数周时间。
- **记录每个智能体的观测和动作流。** 没有每个智能体的跟踪，调试 MARL 是无望的。
- **将训练代码与执行代码分离。** CTDE 是一种纪律；让执行路径真的只看到 `o_i`。
- **奖励塑形警告。** MARL 对奖励设计极其敏感。塑形中的一个协调 bug 就会让智能体学会利用它。运行对抗性测试。
- **对于 LLM 智能体**，先考虑提示词级策略。仅在交互数据 + 奖励信号 + 基础设施都具备时才投资 MARL 训练。

## 练习

1. 运行 `code/main.py`。测量独立和 MAPPO 风格智能体之间的到达目标步数差距。在 6x6 网格上差距是增大还是缩小？
2. 实现竞争变体：两个智能体，一个颗粒，只有第一个到达的获得奖励。哪种模式干净地处理竞争？历史上是 MADDPG。
3. 阅读 MADDPG（arXiv:1706.02275）第 3 节。用你自己的话在伪代码中符号化地实现精确的批评者更新规则。
4. 阅读 MAPPO（arXiv:2103.01955）。为什么作者认为集中价值 + PPO 在他们的基准上击败离策略 MARL？列出三个最强的主张。
5. 将 CTDE 作为设计模式应用于假设的 LLM 智能体系统（例如研究智能体 + 摘要智能体 + 编码智能体）。设计时可用的联合信息中，哪些在运行时不可用？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| MARL（多智能体强化学习） | "Multi-Agent RL" | 多智能体系统的强化学习。 |
| CTDE（集中训练分散执行） | "Centralized Training, Decentralized Execution" | 用全局信息训练；用本地策略部署。 |
| MADDPG | "Multi-Agent DDPG" | 带每个智能体批评者看到所有观测 + 动作的 CTDE。 |
| QMIX | "价值分解" | 每个智能体 Q 值的单调混合。合作型。 |
| MAPPO | "Multi-Agent PPO" | 带集中价值函数的 PPO。2026 年默认基线。 |
| Value decomposition（价值分解） | "个体 Q 值之和" | 联合 Q 表示为每个智能体 Q 值的单调函数。 |
| Non-stationarity（非平稳性） | "移动靶" | 每个智能体的环境随其他智能体学习而变化。MARL 的核心问题。 |
| On-policy / off-policy（在策略/离策略） | "从当前/回放中学习" | PPO 是在策略的（MAPPO）；DDPG 和 Q-learning 是离策略的。 |
| SMAC（星际争霸多智能体挑战） | "StarCraft Multi-Agent Challenge" | 合作微操基准；QMIX 的主场。 |
