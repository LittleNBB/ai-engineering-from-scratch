# 智能体的共识与拜占庭容错（Consensus and Byzantine Fault Tolerance for Agents）

> 经典分布式系统 BFT 遇上了随机性的 LLM。2025-2026 年出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）通过置信度探测对每票加权；**DecentLLMs**（arXiv:2507.14928）采用无领导者架构，工人并行提案并用几何中位数聚合；**WBFT**（arXiv:2505.05103）结合加权投票与层次结构聚类来划分核心和边缘节点。"Can AI Agents Agree?"（arXiv:2603.01213）的诚实实证结果是，即使标量一致性在今天也很脆弱——单个欺骗性智能体就能破坏 Mixture-of-Agents。BFT 是必要的但不充分。本课构建一个最小 BFT 协议，注入三种智能体特定攻击（拜占庭谎言、谄媚从众、相关错误单一种群），并测量每种共识变体的应对能力。

**类型：** Learn + Build（学习 + 构建）
**语言：** Python（标准库）
**前置课程：** Phase 16 · 07（Society of Mind and Debate），Phase 16 · 13（Shared Memory）

## 问题

你有 N 个 LLM 智能体各产生一个答案。它们不一致。多数投票选了错误的那个，因为两个智能体是相关的（相同基础模型、相同训练数据、相同失败模式）。第三个智能体恰好以一种新颖的方式出错——所以多数是虚假多数。

现在添加一个欺骗性智能体：它故意撒谎。或一个谄媚性智能体：它同意最后发言的那位。在经典 BFT 中，假设拜占庭节点是 `f < n/3` 的比例且行为任意。2026 年的现实是，LLM 节点即使在诚实时也是随机的，跨模型相关，并受彼此输出影响。你不能把它们当作独立的伯努利投票者。

经典 BFT（PBFT，1999）没有错——它是不完整的。它处理任意比特翻转。它不处理"三个诚实智能体因为共享训练数据而共享一个幻觉"。本课从 PBFT 的基础出发，叠加三个 2025-2026 年的适应性改进。

## 核心概念

### 经典 BFT 给你什么

实用拜占庭容错（Castro & Liskov，OSDI 1999）容忍 `f < n/3` 个拜占庭节点。协议有三个阶段（预准备、准备、提交）和两个原语（签名消息、法定人数证书）。在 `n >= 3f + 1` 个诚实或恶意节点之间就单个值达成一致。

保证是强的，但假设：

1. **独立故障。** 拜占庭节点不协调。
2. **诚实节点真正诚实。** 诚实输出的正确性不是问题；协议只对齐分歧。
3. **问题有基准真相答案。** 对错误事实的共识仍然是共识。

LLM 智能体违反了所有三个假设。两个运行相同基础模型的智能体共享故障。一个"诚实"的 LLM 仍然会产生幻觉。在模糊问题上，"真相"就是智能体决定的——没有外部预言机。

### 三种 LLM 特定攻击

**拜占庭谎言。** 一个智能体输出故意错误的答案。如果 `f < n/3`，经典 BFT 可以处理。

**谄媚从众。** 一个智能体在投票前阅读其他人的答案，并与最后发言的那位对齐。不是恶意的，但与最大声的声音相关。经典 BFT 不阻止这一点，因为该智能体通过了每个签名检查。

**相关错误单一种群。** 三个智能体共享一个基础模型。它们产生相同的错误幻觉答案。多数是错的。经典 BFT 无济于事，因为三个都"诚实地"同意了。

### 2025-2026 年的应对方案

**CP-WBFT**（arXiv:2511.10400）— 置信度探测加权 BFT。每个投票者对其答案附加一个置信度探测（自报告概率，或单独校准模型的预测）。投票权重随置信度缩放。报告在完全图上 +85.71% 的 BFT 改善。缓解：谄媚从众（从众智能体对其自愿立场往往置信度较低）。

**DecentLLMs**（arXiv:2507.14928）— 无领导者。工人智能体并行提案，评估者智能体评分提案，最终答案是评分位置的几何中位数。在 `f < n/2` 时鲁棒。缓解：拜占庭谎言和相关错误（几何中位数对异常值鲁棒，向密集聚类而非模型偏差平均值拉近）。

**WBFT**（arXiv:2505.05103）— 带层次结构聚类的加权 BFT。投票权重由响应质量加上从历史学习的信任分数分配。将智能体聚类为核心和边缘；核心智能体必须先达成共识，边缘智能体跟随。缓解：可扩展性（核心共识小且快）以及部分缓解单一种群（核心可以选择多样性）。

### 实证："Can AI Agents Agree?"（arXiv:2603.01213）

该论文测量了多个前沿模型上的标量一致性（LLM 智能体就单个数值达成一致）。发现令人不安：

- 即使没有对手，LLM 智能体在许多基准上对标量问题的不一致率超过 30%。
- 一个采用欺骗性人格的智能体可以将 Mixture-of-Agents 共识拉离诚实基线 40+ 个百分点。
- 不一致率与模型多样性相关——异质集成比同质集成不一致更多（好：去相关错误）但漂移更慢（坏：更长的达成一致时间）。

要点：BFT 给你对齐输出的机制，但不告诉你对齐的输出是否正确。结合验证（Phase 16 · 08 角色专业化）、多样性（Phase 16 · 15 辩论变体）和评估者智能体（Phase 16 · 24 基准测试）。

### 核心协议，精简版

LLM 智能体的最小 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信度探测 c_i ∈ [0, 1]
3. 聚合器从所有 n 个智能体收集 (a_i, c_i)
4. 聚合器按语义聚类分组（等价答案）
5. 聚合器计算每个聚类 C 的权重：
     w(C) = sum_{i in C} c_i
6. 获胜者 = 权重最大的聚类，如果 max > threshold * sum(c_i)
   否则：重试或升级
7. 少数聚类带溯源记录以供事后审计
```

语义聚类步骤是 LLM 特定的转折。两个答案"研究报告 4.2%"和"4.2% 的提升"是同一个聚类。朴素的字符串相等检查会遗漏这一点。在生产中，使用廉价的嵌入模型或显式规范化。

### 阈值调优

`threshold` 参数决定何时接受、何时重试。太低：你接受弱势多数。太高：你永远不接受任何东西。经验范围：`n=5-7` 个智能体时 0.5-0.67，更小的 `n` 更高。低于阈值时，升级到人类或不同的智能体集成。

### 共识无济于事的场景

- **模糊问题。** 如果问题没有基准真相，共识是一种意见。就这么称呼它。
- **复合问题。** "写代码并解释它"——两个答案。分别对每个投票。
- **对抗性多轮。** 如果智能体能观察先前轮次并模仿（Du 2023 辩论），它们开始互相同意而不管真相。限制轮次（通常 2-3）。

## 动手实现

`code/main.py` 实现了：

- `AgentVoter` — 带有（答案、置信度）的脚本化策略。
- `MajorityVote` — 经典多数决。
- `CPWBFT` — 带语义聚类的置信度加权投票。
- `DecentLLMs` — 对评分提案的几何中位数聚合。
- `Scenario` — 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`（拜占庭）：一个智能体高置信度撒谎。
2. `sycophancy`（谄媚）：一个智能体复制它看到的第一个答案，匹配置信度。
3. `monoculture`（单一种群）：三个智能体共享一个错误答案（相关错误），中等置信度。

运行：

```
python3 code/main.py
```

预期输出：一个（攻击，聚合器）-> 最终答案的表格，正确答案高亮。多数决在单一种群情况下失败。CPWBFT 的置信度加权缓解谄媚。DecentLLMs 的几何中位数在单一种群不到一半时向诚实聚类拉近。

## 实际应用

`outputs/skill-consensus-designer.md` 为多智能体集成设计共识协议：聚类方法、加权、阈值和低于阈值轮次的升级策略。

## 交付建议

在交付任何共识机制之前：

- **至少用上述三种模式进行攻击测试。** 你的协议应该可预测地失败，而非静默地。
- **记录每个少数聚类及其溯源。** 少数聚类是相关错误的早期预警系统。
- **强制有界轮次。** 不要"持续辩论直到达成一致"——那会奖励谄媚。
- **将一致性与正确性分离。** 共识输出交给验证者；验证者独立于集成。
- **监控一致率。** 急剧上升意味着从众偏差；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认多数决在单一种群攻击下失败，但当单一种群置信度低于 0.7 时 CPWBFT 部分缓解。
2. 添加第四种攻击模式：**静默弃权** — 一个智能体拒绝回答（"我不知道"）。每个聚合器应如何处理弃权？实现你的选择。
3. 将语义聚类从字符串规范化切换为嵌入相似度（使用任何开源嵌入模型）。谄媚攻击会怎样？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探测校准步骤（一个单独的校准模型检查每个智能体的自报告置信度）。测量单一种群场景上的准确率增益。
5. 阅读"Can AI Agents Agree?"（arXiv:2603.01213）。复现一个简化的标量一致性实验：三个智能体、一个标量问题、欺骗性人格提示词。CPWBFT 还是 DecentLLMs 能捕获它？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| BFT（拜占庭容错） | "Byzantine fault tolerance" | Castro-Liskov 1999 协议，容忍 `f < n/3` 个任意故障达成共识。 |
| Byzantine（拜占庭） | "任何不良行为" | 一个可以撒谎、丢弃消息、静默失败的节点——除了安全崩溃以外的任何行为。 |
| Confidence probe（置信度探测） | "你有多确定？" | 附加到投票的自报告或校准器预测的概率。 |
| Semantic clustering（语义聚类） | "相同答案，不同措辞" | 计票前对等价答案进行分组。 |
| Geometric median（几何中位数） | "鲁棒中心" | 最小化到样本点距离之和的点。对异常值鲁棒，不像均值。 |
| Monoculture（单一种群） | "同一个模型，同一个失败" | 当智能体共享训练数据或基础模型时的相关错误。 |
| Sycophantic conformity（谄媚从众） | "同意声音最大的那个" | 智能体的投票偏向于最先/最大声发言的那位。 |
| Core/Edge（核心/边缘） | "层次化 BFT" | WBFT 的划分：小核心先达成共识，边缘节点跟随。限制延迟。 |
