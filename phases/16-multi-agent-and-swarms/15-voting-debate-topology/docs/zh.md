# 投票、自一致性与辩论拓扑（Voting, Self-Consistency, and Debate Topology）

> 最廉价的聚合：采样 N 个独立智能体，多数投票。Wang 等人 2022 年的自一致性用一个模型采样 N 次来做到这一点。多智能体用**异质**智能体扩展它以逃离单一种群——不同模型、不同提示词、不同温度、不同上下文。超越多数投票，辩论拓扑很重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星型/链型/树型/图型协调，发现**图型最适合研究**，超过约 4 个智能体后存在"协调税"。AgentVerse（ICLR 2024）记录了两种涌现模式——志愿行为和从众行为——从众既是特性（发现共识）也是风险（群体思维，Lesson 24）。本课绘制拓扑空间图，构建每个变体，并测量协调税。

**类型：** Learn + Build（学习 + 构建）
**语言：** Python（标准库）
**前置课程：** Phase 16 · 07（Society of Mind and Debate），Phase 16 · 14（Consensus and BFT）
**时间：** ~75 分钟

## 问题

辩论可以提高准确率（Du 等，arXiv:2305.14325）。也可能降低。辩论是否有帮助取决于四个结构选择：

1. 谁和谁说话（拓扑）。
2. 多少轮（Du 2023：轮次和智能体数量都独立地重要）。
3. 智能体是否异质（不同基础模型打破单一种群）。
4. 是否存在对抗性声音（钢铁人论证 vs 稻草人论证）。

直接将"运行 5 个智能体然后投票"附加到任务上的团队往往比单智能体更差。失败不是随机的。它们与拓扑和异质性相关。本课是拓扑图。

## 核心概念

### 自一致性，单模型基线

Wang 等人 2022（"Self-Consistency Improves Chain of Thought Reasoning"）在 temperature > 0 下对同一模型采样 N 次，并对推理路径答案进行多数投票。GSM8K 上的结果：N=40 样本比单次贪心解码有显著增益。自一致性是多智能体投票的单智能体前身。

局限：自一致性使用一个基础模型。错误在构造上就是相关的。如果模型有系统性偏差，所有 N 个样本都共享它。

### 多智能体投票，异质扩展

用 N 个*不同的*智能体替代 N 个样本。不同基础模型（Claude、GPT、Llama）、不同提示词、不同工具访问。好处：去相关错误。代价：不同智能体成本不同；协调它们增加开销。

异质辩论在 2026 年的经典名称是 **A-HMAD** —— 对抗性异质多智能体辩论（Adversarial Heterogeneous Multi-Agent Debate）。并非普遍采用，但论文用这个术语指"不同模型辩论，减少单一种群的相关错误"。

### 四种拓扑

```
星型                链型               树型                图型

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           （全连接）
```

星型：一个中心，所有其他智能体只与中心通信。等价于没有反向通道的监督者-工人。
链型：线性，每个智能体看到前一个的输出。类似流水线。
树型：层次化，用于层次化智能体系统（Lesson 06）。
图型：任意到任意。包括全连接团和任意 DAG。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在包括研究、编码和规划的任务套件上对星型、链型、树型、图型进行了基准测试。关键测量结果：

- **图型**拓扑在研究任务上胜出。信息任意到任意流动；智能体可以互相批评。
- **星型**在快速回答事实性任务上胜出。中心过滤和整合。
- **链型**在逐步流水线（分阶段精炼）上胜出。
- **协调税**在图型拓扑中超过约 4 个智能体后出现。挂钟时间和 token 成本增长快于质量。

4 个智能体的上限是经验性的，不是根本性的。它反映了 2026 年 LLM 上下文容量：每个智能体的上下文填满了对等者的输出，一旦每个人都能看到每个人，添加智能体 N+1 的边际价值就下降了。

### 多智能体辩论策略（"Should we be going MAD?"）

arXiv:2311.17371 是 2023 年的 MAD 策略综述。被他人复现的关键发现：与自一致性*结构相似*的 MAD 变体（独立采样 + 聚合）在相同预算下往往不如自一致性。当智能体真正异质且辩论具有对抗性结构（一个智能体反驳）时，MAD 帮助最大。

### AgentVerse 涌现模式

AgentVerse（ICLR 2024）记录了即使没有显式设计也会从多智能体辩论中涌现的两种行为：

- **志愿行为（Volunteer）。** 一个智能体主动提供帮助（"我可以做下一步"）。有用：它将工作分配给最适合子任务的智能体。
- **从众行为（Conformity）。** 一个智能体调整其立场以匹配批评者，即使批评者是错的。这是辩论版的谄媚（Lesson 14）。

从众是为什么辩论直到达成一致会奖励霸凌者的原因。有界轮次加上单独的裁判可以缓解。

### 异质性：真正移动准确率的旋钮

2024-2026 年实践文献中的一个模式：将你的 N 个智能体中的一个替换为不同的基础模型，比增加 N 到 N+1 带来更大的准确率提升。直觉是单一种群——每个新的独立错误来源比额外的相关样本更有价值。

在极限情况下，异质性胜过数量。三个不同模型在大多数有清晰基准真相的任务上打败五个同一模型的副本。

### 陪审团方法（Jury Methods）

Sibyl 框架（在 Minsky-LLM 文献中被引用）形式化了一个"陪审团"——一组小型专业智能体通过在每个阶段投票来精炼答案。与简单多数投票不同，陪审团有角色：一个智能体交叉询问，一个提供上下文，一个评分合理性。陪审团方法是简单投票（便宜、易受单一种群影响）和完整 MAD（昂贵、易受从众影响）之间的中间点。

### 投票加辩论适用场景

- 问题有基准真相（事实、数学、代码行为）。投票收敛是有意义的。
- 智能体可以访问不同的来源或工具（异质性可用）。
- 轮次有界（通常 2-3），有单独的裁判或验证者。
- 预算允许 3-5 个智能体。在图型拓扑中超过 5-7 个，协调税占主导。

### 投票加辩论有害场景

- 问题是意见型的。智能体收敛到看起来最自信的答案，而非最正确的。
- 所有智能体共享一个基础模型。单一种群使共识毫无意义。
- 轮次无界。从众每次都赢。
- 任务简单。一个使用 N=5 自一致性的单智能体更便宜且一样准确。

## 动手实现

`code/main.py` 实现了：

- `run_star(agents, hub, question)` — 中心轮询每个工人，聚合。
- `run_chain(agents, question)` — 串行精炼。
- `run_tree(root, children, question)` — 带深度-2 聚合的层次化。
- `run_graph(agents, question, rounds)` — 全对全辩论，有界轮次。
- 脚本化的异质性旋钮：每个智能体有一个 `error_bias` 指示其系统性错误。
- 测量工具在 N=3、5、7 下运行每种拓扑并报告（准确率、总 token、模拟挂钟时间）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N →（准确率、token、延迟）的表格。图型在 N=3-5 的研究型任务上胜出；星型在快速事实性任务上胜出；图型在 N=7 时显示协调税（延迟膨胀快于准确率）。

## 实际应用

`outputs/skill-topology-picker.md` 是一个技能，读取任务描述并推荐拓扑（星型/链型/树型/图型）、N（智能体数量）、异质性配置（使用的基础模型）和轮次上限。

## 交付建议

对于任何集成：

- 从使用一个强大基础模型的 **N=5 自一致性**开始。这是廉价基线。
- 如果准确率重要，升级到 **N=3 异质投票**。测量差异。
- 只在任务有结构（研究、多步骤）且有界轮次可行时才升级到**辩论拓扑**。
- 始终记录少数聚类。当少数持续正确时，你有一个多样性信号。
- 在准确率旁边基准测试挂钟时间和 token。"10 倍成本换更好准确率"是一个商业决策。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Self-consistency（自一致性） | "采样 N 次，投票" | Wang 2022。单模型，N 次 temperature>0 采样，对推理路径多数投票。 |
| Heterogeneity（异质性） | "不同模型" | 不同基础模型或提示词家族的集成。打破单一种群。 |
| MAD（多智能体辩论） | "Multi-agent debate" | 智能体在轮次中交换批评的通用术语。见 Du 2023。 |
| A-HMAD（对抗性异质 MAD） | "Adversarial Heterogeneous MAD" | 强调不同模型 + 对抗性结构的 MAD 变体。 |
| Topology（拓扑） | "谁和谁说话" | 星型、链型、树型、图型。决定信息流。 |
| Coordination tax（协调税） | "递减收益" | 在图型中超过约 4 个智能体，成本增长快于质量。 |
| Volunteer behavior（志愿行为） | "主动帮忙" | AgentVerse 涌现模式：一个智能体主动承担一个步骤。 |
| Conformity behavior（从众行为） | "压力下的同意" | AgentVerse 涌现模式：一个智能体与批评者对齐。 |
| Jury（陪审团） | "小型专业小组" | Sibyl 风格的带角色集成（审查者、上下文、评分者）。 |
